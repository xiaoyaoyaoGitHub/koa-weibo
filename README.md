# jwt vs session
- 都是为了解决: 登录&存储登录用户的信息
- jwt 存储在客户端 不依赖cookie, 可跨域(使用的是header, 跨域是针对cookie) 更适合于服务节点比较多, 跨域比较多的系统
- session 存储在服务端 依赖cookie 默认不可跨域  更适合于统一的web服务,server要严格管理用户信息
